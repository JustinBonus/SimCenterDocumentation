pelicun

Performs damage and loss (DL) assessment using the pelicun Python package. The application uses built-in fragility functions and the replacement cost defined in the building source file to determine damage state and loss likelihoods.

Inputs
CLI
DL file <string>
File containing the DL model information. The default is to source it from the building source file.
EDP file <string>
File containing type, location, and direction of recorded EDPs from the response simulation.
DL_Method <string>
Defines the damage and loss assessment method. Available choices include 'FEMA P58', 'HAZUS MH EQ', 'HAZUS MH', 'HAZUS MH EQ IM', and 'HAZUS MH HU'.
Realizations <int>
Specifies the number of loss realizations to generate.
detailed_results <bool>
If False, only the main results are saved to improve efficiency.
log_file <bool>
If False, pelicun does not create a run log file.
coupled_EDP <bool>
If True, the raw EDPs are used as-is without resampling.
event_time <string>
Defines the time of the event in YYYY-MM-DD:HH format. 'off' turns all time-effects off.
ground_failure <bool>
If True, fragility groups associated with ground failure are added in the auto-population phase. Note that such analysis requires PGD values as inputs.
auto_script <string>
